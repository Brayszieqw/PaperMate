# paper-writer 与中国大学生论文 / 报告流程对照分析

> 目的：检查当前 `paper-writer` 的内部阶段与运行步骤，是否贴近中国大学生写论文 / 写课程报告 / 毕业论文的真实流程，并找出还缺哪些关键环节。

## 1. 结论先行

当前 `paper-writer` 的主干流程：

- `paper-scout`
- `paper-library`
- `paper-drafter`
- `paper-research-ops`
- `paper-reviewer`

已经能覆盖中国大学生写论文 / 报告过程中的**文献搜集、笔记整理、草稿生成、局部精读、文字审查**这几块核心工作。

但如果对照更真实的高校论文流程，仍然缺少几类非常常见且很重要的环节：

1. **选题与题目收束**
2. **提纲 / 开题 / 方案论证**
3. **答辩准备 / 问答防守**
4. **格式规范与学校模板合规**
5. **版本修订与导师反馈吸收**

换句话说：

当前 `paper-writer` 更像一个“读文献 + 写草稿 + 做审查”的智能体，
但离“中国学生真实论文流程助手”还差几个贴近院校实践的关键步骤。

## 2. 中国大学生常见论文 / 报告流程（抽象版）

虽然不同学校、不同学院要求不完全一致，但通常可抽象为：

```text
1. 选题 / 收束方向
2. 查资料 / 找文献
3. 做笔记 / 形成问题意识
4. 写提纲 / 开题 / 方案说明
5. 写初稿
6. 反复修改
7. 核对引用 / 降重 / 检查逻辑
8. 调格式 / 模板合规
9. 准备答辩 / 准备汇报
10. 面对老师提问进行说明
```

如果是课程报告，流程会短一些；如果是本科毕业论文，流程会更完整、更正式。

## 3. 当前 paper-writer 各阶段与学生流程的对应关系

## 3.1 `paper-scout`

### 对应现实流程
- 查资料 / 找文献
- 建立候选文献池

### 对应程度
- **高匹配**

### 说明
这部分和学生真实需求非常贴近。学生写论文首先就会遇到：
- 这个方向该搜什么
- 哪些论文值得先看
- 哪些可以先跳过

`paper-scout` 目前承担得比较合理。

## 3.2 `paper-library`

### 对应现实流程
- 做笔记
- 整理文献
- 形成概念索引

### 对应程度
- **高匹配**

### 说明
中国学生常见问题之一就是：
- 看了很多论文，但没有形成结构化资料

`paper-library` 这块非常有价值，尤其适合避免“读了就忘”。

## 3.3 `paper-drafter`

### 对应现实流程
- 写提纲
- 写初稿
- 写章节草稿

### 对应程度
- **中高匹配**

### 说明
它已经覆盖了“写”的核心，但还偏向：
- 把材料变成草稿

而没有完全覆盖：
- 中国学生常见的“老师要求我先给目录 / 提纲 / 开题说明”的场景

## 3.4 `paper-research-ops`

### 对应现实流程
- 精读单篇论文
- 翻译术语 / 段落
- 看方法细节 / 实验设置

### 对应程度
- **中高匹配**

### 说明
这非常符合学生卡住时的现实需求：
- 这篇论文到底讲了什么
- 这段怎么翻
- 这个实验到底怎么做的

## 3.5 `paper-reviewer`

### 对应现实流程
- 核对逻辑
- 检查引用
- 看有没有过度结论
- 修改前自查

### 对应程度
- **高匹配**

### 说明
这部分对中国学生尤其重要，因为很多论文不是“不会写”，而是：
- 写出来经不起问
- 写出来逻辑松
- 写出来引用撑不住

## 4. 当前缺失的关键环节

## 4.1 缺：选题 / 题目收束阶段

### 为什么重要
中国学生做论文最常见的第一卡点，不是写正文，而是：
- 题目太大
- 题目太空
- 研究问题不清楚

### 当前状态
`paper-writer` 虽然能“帮助研究”，但没有一个明确阶段专门处理：
- 题目压缩
- 研究问题收束
- 可行性判断

### 建议补法
不一定要再暴露新 agent，可以在单入口里新增一个内部阶段标签：

- `topic-framing`

主要负责：
- 题目收束
- 研究问题定义
- 边界缩小
- 为什么选这个方向

## 4.2 缺：开题 / 提纲 / 方案论证阶段

### 为什么重要
中国高校里很常见：
- 老师先看提纲
- 先交开题报告
- 先说明研究思路和方法

### 当前状态
`paper-drafter` 部分覆盖提纲，但没有明确把“开题式论证”单独凸显出来。

### 建议补法
在 `paper-drafter` 之上或之内，显式支持一个：

- `proposal-outline` / `opening-report` 模式

让系统能优先生成：
- 研究背景
- 研究意义
- 问题定义
- 方法思路
- 预期贡献

## 4.3 缺：答辩准备阶段

### 为什么重要
这正是你刚提出的重点。

学生真正焦虑的经常不是“写出来”，而是：
- 老师问为什么选这个题
- 老师问为什么方法这样设计
- 老师问为什么不用别的方法
- 老师问这一条结论凭什么成立

### 当前状态
我们已经引入 `defense-ready lite`，但还没有一个明确、系统化的答辩准备阶段。

### 建议补法
在单入口内部新增一个关键阶段标签：

- `defense-prep`

它可以负责：
- 可能问题清单
- 每个问题的回答提纲
- 研究方向 / 方法选择 / 实验设计的解释链
- “如果老师追问”的二层回答

## 4.4 缺：格式规范 / 模板合规阶段

### 为什么重要
中国大学生论文一个现实特征是：
- 很多问题不是内容，而是格式
- 标题编号、参考文献格式、学校模板、摘要体例等要求很多

### 当前状态
目前 `paper-writer` 更偏内容工作流，不太覆盖格式与模板合规。

### 建议补法
不一定需要新 agent，但建议未来补一个内部阶段：

- `format-check`

负责：
- 模板合规
- 参考文献格式
- 中文摘要 / 英文摘要检查
- 图表题注 / 编号一致性

## 4.5 缺：导师反馈吸收与修订闭环

### 为什么重要
中国学生写论文不是一次写完，而是：
- 老师批一版
- 改一版
- 再批
- 再改

### 当前状态
我们有 review，但更像系统自检；还没有把“导师反馈吸收”当成独立现实场景来建模。

### 建议补法
未来可新增或内化一个：

- `revision-loop`

负责：
- 吸收导师意见
- 把反馈映射到具体章节
- 跟踪已改 / 未改项

## 5. 建议的“更贴近中国学生”的内部阶段结构

在不破坏“单可见智能体”前提下，建议把内部阶段从当前 5 个扩展成更贴近学生流程的 8 个：

```text
paper-writer internal stages
1. topic-framing      # 选题收束 / 问题定义
2. paper-scout        # 搜文献 / 候选池
3. paper-library      # 笔记 / 入库 / 索引
4. proposal-outline   # 提纲 / 开题 / 方案论证
5. paper-drafter      # 正文草稿
6. paper-research-ops # 精读 / 翻译 / 方法细节
7. paper-reviewer     # 引用 / claim / 风险审查
8. defense-prep       # 答辩准备 / 追问防守
```

如果后续再扩展，可附加：

- `format-check`
- `revision-loop`

## 6. 哪些是“必须补”的，哪些可以后补

## 必须优先补

### 1. `topic-framing`
因为很多学生一开始就卡在题目和问题边界上。

### 2. `defense-prep`
因为这正是你明确提出的目标，而且它能直接提高整套系统的“可回答性”。

### 3. `proposal-outline`
因为它最贴近中国高校“先看提纲 / 先做开题”的真实流程。

## 可后补

### 4. `format-check`
重要，但可以放到内容链更稳之后。

### 5. `revision-loop`
非常实用，但可等真实反馈闭环出现后再补。

## 7. 对当前体系的建议

### 短期建议
先不要重新暴露多个 agent，而是在单入口里把以下三个阶段显式内化：

- `topic-framing`
- `proposal-outline`
- `defense-prep`

### 中期建议
让 routePacket 能识别这三类学生真实目标：

- “帮我缩题 / 收束方向”
- “帮我写提纲 / 开题报告”
- “帮我准备答辩 / 预测老师提问”

### 长期建议
再补：

- `format-check`
- `revision-loop`

## 8. 一句话收束

> 当前 `paper-writer` 已经很像“科研资料与写作智能体”，但若要更贴近中国大学生真实论文流程，还需要补上“选题收束、开题提纲、答辩准备”这三块最具院校实践特征的阶段能力。
